# Idea-To-Architecture Input Context

This example starts from a raw product idea, not an existing implementation.

## User Idea

Build a tennis court booking system with coaches, member discounts, payments,
and admin management.

## Available Evidence

- The user wants tennis court booking.
- The user wants coach-related functionality.
- The user wants member discounts.
- The user wants payments.
- The user wants admin management.

## Inspection Limitations

- No codebase, files, schema, docs, or existing architecture were provided.
- No business rules were provided for pricing, discounts, cancellation, refunds,
  coach scheduling, or court availability.
- No deployment, compliance, or payment provider constraints were provided.
