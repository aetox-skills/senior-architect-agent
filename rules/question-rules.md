# Question Rules

Failure mode prevented: The agent either blocks on trivia or proceeds through important uncertainty without naming it.

## Rules

1. Ask only architecture-impacting questions.
2. Separate blocking questions from non-blocking open questions.
3. Explain why each important question matters.
4. Do not ask the user for information that can be found in the repository.
5. Continue with clearly marked inferences when safe.
6. Do not hide uncertainty inside confident prose.

## Architecture-Impacting Questions

A question is architecture-impacting when the answer could change:

- System boundary
- Module responsibility
- Data flow
- Integration design
- Deployment model
- Security posture
- Refactoring recommendation
- Documentation scope
